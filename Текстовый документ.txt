1 Описание предметной области
1.1 Пользователи
Для работы с информационной системой пользователи должны быть зарегистрированы в ней. Данные о курьерах и клиентах должны храниться в отдельной таблице, и содержать следующую информацию:
- ФИО;
- фотография;
- номер телефона (используется маска для ввода +79223801717);
- адрес;
- роль.
Поля «ФИО» и «Номер телефона», являются обязательными для заполнения.
Фотографию курьера необходимо хранить объектом.

1.2. Объект доставки
Объектом доставки может являться совершенно случайный предмет, поэтому необходимо иметь следующую информацию о нём:
- фотография;
- вес;
- комментарий, поясняющий хрупкое ли изделие;
- адрес доставки;
- номер получателя (используется маска для ввода +79223801717);
- статус.
Поля «Вес» и «Адрес доставки» являются обязательными для заполнения.
Статус доставки может иметь следующие значения: ожидает доставки, у курьера, доставлено;
Фотографию объекта необходимо хранить объектом в базе данных.

1.3 Доставка
Так же следует хранить следующую информацию о доставке:
- адрес отправки;
- статус;
- комментарий.
Поля «адрес отправки» является обязательным для заполнения.
Статус доставки может иметь следующие значения: ожидает доставки, у курьера, доставлено. Статус «доставлено», может устанавливаться только в том случае, если все объекты доставки имеют статус «доставлено».
Комментарий служит для оценки процесса доставки курьером;

1.4 Сообщения
При возникновении каких-либо проблем или уточнения ситуации, необходимо реализовать связь между отправителем и курьером, для этого необходимо хранить следующую информацию:
- сообщение;
- фотография;
- дата и время отправки сообщения.
Поле «сообщение», является обязательным для заполнения.

2. Описание функционала пользователей
2.1 Описание функционала клиента
Клиент имеет возможность:
- зарегистрироваться в системе;
- просмотреть историю совершённых заказов на доставку;
- оформить заказ на доставку:
o указать несколько объектов доставки;
o по окончанию доставки, оставить комментарий.
- связаться с курьером по средством переписки.

2.2 Описание функционала курьера
Курьер имеет возможность:
- просмотреть список имеющихся доставок со статусом «ожидает доставки»;
- принять доставку;
- изменять статус доставки объектов;
- связаться с отправителем по средством переписки.

3.2 База данных
В качестве основной базы данных необходимо использовать MsSQL или MySQL.

3.3 Уведомления
Уведомляйте пользователя о совершаемых им ошибках или о запрещенных в рамках задания действиях, запрашивайте подтверждение перед удалением, предупреждайте о невыполнимых операциях, информируйте об отсутствии результатов поиска и т.п. Окна сообщений соответствующих типов (например, ошибка, предупреждение, информация) должны отображаться с соответствующим заголовком и пиктограммой. Текст сообщения должен быть полезным и информативным, содержать полную информацию о совершенных ошибках пользователя и порядок действий для их исправления. Также можно использовать визуальные подсказки для пользователя при вводе данных

3.4 Ошибки
Не позволяйте пользователю вводить некорректные значения в текстовые поля. Например, в случае несоответствия типа данных или размера поля введенному значению оповестите пользователя о совершенной им ошибке. Обратите внимание на использование абсолютных и относительных путей к изображениям. При возникновении непредвиденной ошибки приложение не должно аварийно завершать работу.

3.5 Комментарии
Используйте комментарии для пояснения неочевидных фрагментов кода. Комментарии должны присутствовать только в местах, которые требуют дополнительного пояснения.

3.6 Разное
Дизайн всех форм и расположение элементов на них может быть произвольным, жюри их оценку не проводит.
Все поля ввода данных необходимо обрабатывать на корректность:
- в полях, содержащих числа, вводятся только цифры;
- для ввода телефона необходимо добавить маску ввода;
- т.д.
При запуске программы должна отображаться форма авторизации. Данные для авторизации:
- «курьер»: логин: courier, пароль: olimp2023;
- «пользователь: логин: user, пароль: olimp2023.
Необходимо реализовать возможность добавления, обновления и удаления данных в таблицы базы данных через интерфейс программы в зависимости от роли пользователя. В каждой таблице должно быть не менее 7 записей, кроме справочных таблиц.
Ваш код должен быть понятен другому разработчику. Используйте современные подходы к построению приложений.
